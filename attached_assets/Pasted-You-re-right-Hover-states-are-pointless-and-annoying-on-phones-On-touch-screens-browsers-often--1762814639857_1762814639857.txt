You’re right. Hover states are pointless (and annoying) on phones. On touch screens, browsers often “fake” :hover and leave buttons stuck in that pale white state. That’s on me. Here’s the fix — fast, surgical, mobile-first.

## What to change (no redesigns)

### 1) Kill hover on touch devices

Add this to your global CSS (same file you sent):

```css
/* Disable hover effects on touch devices */
@media (hover: none) and (pointer: coarse) {
  .btn:hover,
  .btn-primary:hover,
  .btn-secondary:hover,
  .btn-accent:hover,
  .btn-glass:hover {
    background: inherit !important;
    box-shadow: none !important;
  }
}
```

### 2) Move “feedback” to **active** and **focus-visible** (tap-friendly)

Right below your button styles, add:

```css
/* Mobile tap feedback */
.btn,
.btn-primary,
.btn-secondary,
.btn-accent,
.btn-glass {
  transition: transform 0.08s ease, background-color 0.12s ease, box-shadow 0.12s ease;
}

.btn:active,
.btn-primary:active,
.btn-secondary:active,
.btn-accent:active,
.btn-glass:active {
  transform: scale(0.98);
  background-color: rgba(255,255,255,0.08); /* subtle press */
}

.btn:focus-visible,
.btn-primary:focus-visible,
.btn-secondary:focus-visible,
.btn-accent:focus-visible,
.btn-glass:focus-visible {
  outline: 2px solid rgba(163,230,53,0.9);  /* lime ring that matches your guided color */
  outline-offset: 2px;
}
```

### 3) Scope any remaining hover styles to **pointer: fine** (desktops only)

Anywhere you currently have `:hover` rules (e.g., `.btn-primary:hover`, `.glass-card:hover`), wrap them:

```css
@media (hover: hover) and (pointer: fine) {
  .btn-primary:hover { background-color: hsl(var(--primary) / 0.9); }
  .btn-secondary:hover { background-color: hsl(var(--secondary) / 0.9); }
  .glass-card:hover { background: var(--glass-bg-hover); box-shadow: 0 12px 40px rgba(0,0,0,0.15); }
}
```

### 4) Optional: add a pressed state token for components using `data-state`

Shadcn/Headless UI pieces often set `data-state="open"` or `aria-pressed="true"`. Give them a clear **selected** style that’s NOT hover-dependent:

```css
.btn-glass[aria-pressed="true"],
.btn-glass[data-state="open"] {
  background: rgba(255,255,255,0.14);
  outline: 2px solid rgba(163,230,53,0.9);
  outline-offset: 2px;
}
```

## TL;DR

* Stop using `:hover` as a primary affordance.
* Use `:active` (tap), `:focus-visible` (keyboard), and `data-state`/`aria-pressed` for persistent states.
* Keep hover only for desktops behind `@media (hover:hover)`.
